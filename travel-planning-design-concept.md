# 旅遊規劃設計概念

> 如何將一趟複雜的多國自由行，設計成清晰、可執行、易於分享的規劃產出物。

---

## 一、設計原則

### 1.1 資訊層次優先於資訊量

旅遊規劃最常見的問題不是資訊不夠，而是資訊太多卻沒有層次。一份好的行程規劃需要回答三個不同層次的問題：

- **策略層**：這趟旅行的路線邏輯是什麼？為什麼這樣走？
- **戰術層**：每天去哪裡、怎麼移動、住在哪個區域？
- **執行層**：哪些票要提前買、哪些要注意、移動需要多少時間？

三個層次分開呈現，讀者才能在需要的時候找到需要的資訊，而不是在一大段文字裡自己篩選。

### 1.2 動線邏輯優先於景點清單

景點清單是最低階的規劃形式。真正有價值的規劃是**路線邏輯**——為什麼這個城市接那個城市、為什麼這個順序走最省時間、哪些地方可以合併、哪些地方必須分開住。

本次伊比利半島行程採用「單向環形」路線設計：

```
法國進 → 葡萄牙北上 → 西班牙南部橫切 → 馬德里 → 巴塞隆納出
```

這條動線的核心優勢：不走任何回頭路，每一段移動都是在前進，不需要額外的交通成本來「歸位」。

### 1.3 限制條件是設計的起點

旅遊規劃是有限資源的最佳化問題。關鍵限制條件包括：

- **固定天數**：這次 16 天，決定了能涵蓋的城市數量上限
- **固定入出境**：CDG 入、BCN 出，決定了路線方向
- **門票稀缺性**：阿爾罕布拉宮的票限量，決定了格拉納達必須在行程中，且不能是最後一刻才決定
- **季節時機**：11 月淡季的人潮與氣候，決定了景點優先順序（夏天排不到的，11 月反而更容易）

好的規劃是把這些限制條件列出來，**讓限制條件驅動設計決策**，而不是設計完了再來發現限制。

## 二、城市選擇邏輯

### 2.1 入選標準

每個城市的入選基於以下至少兩項條件：

| 條件 | 說明 |
|------|------|
| 世界級不可替代景點 | 該城市有其他地方找不到的經典，例如阿爾罕布拉宮、聖家堂 |
| 文化體驗獨特性 | 格拉納達的免費 Tapas 文化、波多的波特酒窖，是只有在當地才能感受的體驗 |
| 路線動線合理性 | 城市間的地理位置能串成順暢的移動路徑，無需繞路 |
| 停留天數可行性 | 在總天數限制內，該城市的核心景點能在分配的天數內完成 |

### 2.2 捨棄南法的理由

原始規劃曾包含南法（亞維儂、尼斯），最終捨棄，原因如下：

- 南法加入後，葡萄牙壓縮至 2 天，無法完整體驗里斯本 + 辛特拉 + 波多
- 格拉納達只剩半天，阿爾罕布拉宮無法完整參觀（納斯里德宮 + 堡壘 + 花園需要完整一天）
- 南法在概念上屬於「法國行程」而非「伊比利半島行程」，混入會稀釋主題聚焦

**設計取捨原則：寧可每個地方玩深，不要每個地方走馬看花。**

### 2.3 城市停留天數分配

| 城市 | 天數 | 分配邏輯 |
|------|------|----------|
| 巴黎 | 3 天 | 三大博物館（羅浮宮、奧塞、凡爾賽）各需半天，蒙馬特補充 |
| 里斯本 | 3 天 | 阿爾法馬 + 貝倫各需一天，辛特拉近郊另需一天 |
| 波多 | 2 天 | 核心景點密集，2 天已足夠深度體驗 |
| 塞維亞 | 2 天 | 王宮 + 大教堂一天、廣場 + 佛朗明哥一天 |
| 格拉納達 | 2 天 | 阿爾罕布拉宮必須一整天，阿爾拜辛另需一天 |
| 馬德里 | 2 天 | 皇宮 + 普拉多一天、蘇菲亞 + 托雷多一天 |
| 巴塞隆納 | 2 天 | 聖家堂 + 奎爾公園一天、蘭布拉 + 哥德區一天 |

## 三、視覺化設計

### 3.1 互動式行程卡的設計思路

靜態的行程表有一個根本缺陷：當行程超過 5 天，使用者很難快速定位「我今天在哪裡、我去過哪些地方、我還沒去哪裡」。

互動式行程卡解決這個問題的方式是**城市篩選（filter by city）**：

- 使用者可以只看某個城市的天數
- 移動段（交通）在城市篩選下隱藏，只在「全部」模式下顯示
- 每個天數卡片保持結構一致（日期、城市標籤、標題、景點、備注），讓視覺掃描快速

### 3.2 資訊密度控制

每張日程卡的內容遵循「三層結構」：

```
標題（一句話說明這天在做什麼）
  └── 景點流水（上午→下午→晚上的順序描述）
        └── 備注（訂票提醒、交通細節、注意事項）
```

備注欄分三種語義顏色：
- **中性灰底**：一般提示（住宿建議、時間技巧）
- **琥珀底（warn）**：需要注意的事項（假日、人潮、特殊狀況）
- **珊瑚底（alert）**：必須行動的事項（提前購票、截止時間）

### 3.3 訂票清單的設計

訂票清單不是附錄，而是**行動系統**。設計重點：

- 按「緊急程度」而非「城市順序」排序
- 每項包含：時間範圍、景點名稱、具體行動、官網連結
- 視覺上用顏色 badge 編碼緊急程度（紅→橙→綠→藍）

這樣的設計讓使用者在決定要旅行後，可以立刻知道「現在該做什麼」，而不是在整份行程裡自己找購票資訊。

## 四、交通規劃邏輯

### 4.1 跨城市交通選擇矩陣

選擇城市間交通工具時，依照以下優先順序評估：

```
時間 × 費用 × 便利性
```

| 情境 | 首選 | 備選 |
|------|------|------|
| 距離 < 400km、有高鐵 | AVE / Alfa Pendular 高鐵 | 長途巴士 |
| 距離 > 600km、無直達高鐵 | 廉價航空 | 高鐵轉乘 |
| 跨國（西班牙→葡萄牙） | 廉航（省 7–8 小時） | ALSA 夜巴 |
| 安達魯西亞城市間 | ALSA 巴士 | 高鐵（班次少） |

### 4.2 入出境點選擇邏輯

「不同點入出境」是多國行程最重要的設計決策：

- **省去回頭路**：若從 BCN 入 BCN 出，西班牙南部來回需多一趟 BCN→安達魯西亞→BCN
- **增加總覆蓋範圍**：CDG 入、BCN 出，讓法、葡、西三國都能納入一趟旅程
- **機票費用通常相近**：來回同點 vs. 不同點入出，價差通常在 €50 以內

### 4.3 城市內交通原則

- **優先步行**：歐洲古城核心景點密集，步行往往比搭車快
- **地鐵為主**：巴黎、馬德里、巴塞隆納地鐵網路完整，購買多日票最划算
- **巴士補充**：里斯本、波多景點較分散，搭巴士或 Tuk-tuk 補充
- **廉航備用**：城市間距離超過 600km 且無直達高鐵時，廉航是最佳選擇

## 五、景點取捨原則

### 5.1 「不可替代性」篩選

每個城市的景點依不可替代程度分三級：

**S 級（必去，不去會後悔）**

這些景點在全球獨一無二，無法在其他地方看到相同的東西：

- 阿爾罕布拉宮（格拉納達）：世界上最完整的摩爾宮殿群
- 聖家堂（巴塞隆納）：高第終身未完成的建築傑作，2026 年即將完工
- 傑羅尼摩斯修道院（里斯本）：曼努埃爾式建築極致，哥倫布時代的歷史現場
- 佛朗明哥表演（塞維亞）：在發源地看佛朗明哥，與旅遊表演截然不同

**A 級（強烈推薦，行程允許必排）**

- 辛特拉佩納宮、馬德里王宮、塞維亞王宮、凡爾賽宮
- 普拉多美術館、奧塞美術館
- 波多里貝拉區、阿爾法馬老城

**B 級（有時間再安排）**

- 近郊一日遊（托雷多、塞哥維亞）
- 購物（La Roca Village、Zara 總部）
- 海灘（巴塞隆內塔）

### 5.2 「體驗層次」設計

一趟旅程應包含至少三種體驗層次，避免只有「看景點」：

| 層次 | 說明 | 範例 |
|------|------|------|
| **視覺奇觀** | 讓人震撼的建築、自然、藝術 | 聖家堂、阿爾罕布拉宮 |
| **文化沉浸** | 融入在地生活方式 | 格拉納達 Tapas 酒吧、波多波特酒窖品酒 |
| **歷史敘事** | 理解當地歷史脈絡 | 里斯本大航海時代、塞維亞摩爾遺跡 |
| **感官體驗** | 食物、音樂、氣味 | 法朵音樂晚餐、佛朗明哥表演、Pastéis de Belém |

## 六、季節時機策略

### 6.1 11 月淡季的優劣分析

**優勢**

- 人潮比夏季（7–8 月）少 40–50%，熱門景點不需要排隊 1–2 小時
- 氣候舒適：葡萄牙 16–20°C、安達魯西亞 18–22°C，不需要夏天防曬
- 住宿與廉航票價相對便宜，品質相同的飯店可能省 30–40%
- 阿爾罕布拉宮等熱門票較容易搶到（但仍需提前）

**劣勢**

- 日照短，下午 17:30 天黑，部分戶外景點要提早安排
- 部分海灘活動與夏季節慶不再舉辦
- 11月1日 法定假日，部分商店休息（但景點大多照常）

**結論**：對文化歷史型旅遊者，11 月是最佳時機之一。對海灘渡假型旅遊者，選夏季更合適。

### 6.2 假日避開策略

西班牙、葡萄牙在宗教節日（聖周、萬聖節、聖誕節）期間：

- 住宿漲價 2–3 倍，且早早售罄
- 塞維亞聖周（Semana Santa）是西班牙最壯觀的節日，但也是住宿最難訂的時刻
- 本次行程 11/1 萬聖節，提前確認每個景點的開放狀態即可

### 7.2 設計可複用性

本次規劃的架構（路線邏輯 → 城市選擇 → 天數分配 → 景點取捨 → 交通矩陣）可直接套用到其他多國行程，例如：

- 義大利環遊（羅馬進、威尼斯出）
- 中歐行程（維也納進、布拉格出）
- 北歐行程（哥本哈根進、斯德哥爾摩出）

核心方法不變：**先確定動線邏輯，再填入城市，再篩選景點，最後規劃交通。**
